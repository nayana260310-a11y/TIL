### Context
개인 비서 서비스와 다수 커뮤니티 소통은 서로 다른 요구사항을 가집니다. 개인 비서 서비스는 프라이버시, 실시간 알림, 파일 전송, 개인화된 상호작용이 중요하며, 다수 커뮤니티 소통은 서버 커스터마이징, 음성 채널, 역할 기반 접근 제어, 대규모 사용자 관리가 필요합니다. n8n을 활용한 자동화 전략 수립 시, 각 플랫폼의 강점을 고려하여 텔레그램을 개인 비서 서비스에, 디스코드를 커뮤니티 소통에 표준으로 적용하는 것이 효율적입니다.

### Core
n8n 워크플로우 예시:

**개인 비서 서비스 (텔레그램)**
```json
{
  "nodes": [
    {
      "parameters": {
        "updates": ["message"],
        "additionalFields": {}
      },
      "name": "Telegram Trigger",
      "type": "n8n-nodes-base.telegramTrigger",
      "typeVersion": 1,
      "position": [250, 300]
    },
    {
      "parameters": {
        "operation": "sendMessage",
        "chatId": "={{$json.chat.id}}",
        "text": "안녕하세요, 개인 비서입니다. 요청하신 작업을 처리 중입니다.",
        "additionalFields": {}
      },
      "name": "Telegram Send Message",
      "type": "n8n-nodes-base.telegram",
      "typeVersion": 1,
      "position": [450, 300]
    }
  ],
  "connections": {
    "Telegram Trigger": {
      "main": [
        [
          {
            "node": "Telegram Send Message",
            "type": "main",
            "index": 0
          }
        ]
      ]
    }
  }
}
```

**커뮤니티 소통 (디스코드)**
```json
{
  "nodes": [
    {
      "parameters": {
        "resource": "channel",
        "operation": "get",
        "channelId": "YOUR_CHANNEL_ID"
      },
      "name": "Discord Get Channel",
      "type": "n8n-nodes-base.discord",
      "typeVersion": 1,
      "position": [250, 300]
    },
    {
      "parameters": {
        "resource": "message",
        "operation": "create",
        "channelId": "YOUR_CHANNEL_ID",
        "content": "새로운 커뮤니티 알림이 도착했습니다!",
        "additionalFields": {}
      },
      "name": "Discord Create Message",
      "type": "n8n-nodes-base.discord",
      "typeVersion": 1,
      "position": [450, 300]
    }
  ],
  "connections": {
    "Discord Get Channel": {
      "main": [
        [
          {
            "node": "Discord Create Message",
            "type": "main",
            "index": 0
          }
        ]
      ]
    }
  }
}
```

### Insight
텔레그램은 개인화된 상호작용과 프라이버시 보호에 강점이 있어 개인 비서 서비스에 적합합니다. 반면, 디스코드는 서버 커스터마이징과 대규모 사용자 관리 기능이 뛰어나 커뮤니티 소통에 최적화되어 있습니다. n8n은 두 플랫폼 모두 공식 노드를 지원하여 유연한 자동화 워크플로우 구축이 가능합니다. 따라서, 개인 비서 서비스에는 텔레그램을, 커뮤니티 소통에는 디스코드를 표준으로 적용하는 것이 기술적 효율성과 사용자 경험을 극대화하는 전략입니다.

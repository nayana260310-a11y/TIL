### Context

파이썬에서 데이터를 필터링할 때 리스트 컴프리헨션(List Comprehension)을 주로 사용하지만, 조건문이 여러 개 중첩되거나 복잡해지면 가독성이 급격히 떨어지는 문제가 발생합니다. 예를 들어 `[x for x in data if condition_a(x) and condition_b(x) and condition_c(x)]` 형태는 이해하기 어렵습니다. 이 경우 가독성을 위해 `filter()` 함수를 사용하는 것이 나을까요? 아니면 성능을 위해 컴프리헨션을 고수해야 할까요? 대량 데이터 처리 시 두 방식의 성능 차이와 가독성 간의 균형을 고민하며 가이드라인을 수립해야 합니다.

### Core

리스트 컴프리헨션과 `filter()` 함수의 문법 및 성능 비교 예시입니다.

**1. 복잡한 조건문 처리 비교**

```python
# 리스트 컴프리헨션 (가독성 저하 가능성)
result_comp = [x for x in data if x > 10 and x % 2 == 0 and x < 100]

# filter() 함수 사용 (가독성 향상 가능성)
result_filter = list(filter(lambda x: x > 10 and x % 2 == 0 and x < 100, data))

# 가독성을 위해 별도 함수 정의 (추천)
def is_valid(x):
    return x > 10 and x % 2 == 0 and x < 100

result_func = [x for x in data if is_valid(x)]
```

**2. 성능 테스트 (대량 데이터 기준)**

일반적으로 리스트 컴프리헨션이 `filter()` + `lambda` 보다 약간 더 빠르거나 유사한 성능을 보입니다. `filter()` 는 함수 호출 오버헤드가 발생할 수 있습니다.

```python
import timeit

data = list(range(1000000))

# 리스트 컴프리헨션
comp_time = timeit.timeit(lambda: [x for x in data if x > 500000], number=100)

# filter + lambda
filter_time = timeit.timeit(lambda: list(filter(lambda x: x > 500000, data)), number=100)

print(f"List Comprehension: {comp_time:.4f} sec")
print(f"Filter + Lambda: {filter_time:.4f} sec")
```

### Insight

*   **성능 측면**: 리스트 컴프리헨션은 내부적으로 최적화되어 있어 `filter()` + `lambda` 조합보다 일반적으로 10~20% 정도 빠르거나 최소한 동등한 성능을 보입니다. `filter()` 는 람다 함수 호출 시 추가적인 오버헤드가 발생할 수 있습니다.
*   **가독성 측면**: 조건이 단순하다면 리스트 컴프리헨션이 가장 직관적입니다. 하지만 조건이 3 개 이상 중첩되거나 논리 연산이 복잡하다면, 별도 함수(`is_valid`) 를 정의하여 컴프리헨션 내부에 호출하는 방식이 가장 가독성이 높습니다.
*   **가이드라인 제안**:
    *   **비전공자/초급 개발자**: 복잡한 조건문은 별도 함수로 분리하여 가독성을 최우선으로 합니다.
    *   **성능 최적화 필요 시**: 대량 데이터 처리 시에는 리스트 컴프리헨션을 사용하되, 조건 함수는 인라인이 아닌 별도 함수로 정의하여 유지보수성과 성능을 모두 잡습니다.
    *   **결론**: `filter()` 는 함수형 프로그래밍 스타일을 선호하는 팀이 아니라면, 가독성 개선을 위해 별도 함수 정의와 함께 리스트 컴프리헨션을 사용하는 것이 파이썬의 'Pythonic'한 방식에 더 부합합니다.
